===================================
1.概要
===================================
ゲームのメニュー画面を表示する処理です。
選択肢を分類毎に一定の列数で改行し、罫線、色分けで視認性を確保します。
また、各コマンドは一定の文字数間隔で行方向に並べるため、コマンドが綺麗に整列されます。

===================================
2.課題
===================================
このゲームエンジンのUIはコマンドプロンプト形式のウインドウで動作するCUIとなっています。

プロンプト形式のCUIの弱点として下記が挙げられます。
①コマンドが多くなると改行が非常に多くなり、スクロール無しで情報を収めるのが困難。
②文字の羅列になるため、一目でどのコマンドが重要か分かりづらい。
③コマンドに絵がないため、プレイヤーがコマンドの分類や性質を把握できない。

メインメニュー画面はプレイヤーが最もよく見る画面となるため、この画面を煩雑にするのはUXを損ね、プレイヤーの離脱を招いてしまいます。
以上のことから、課題解決の重要度が高くなっておりました。

===================================
3.解決方法
===================================
これらの問題を解決できるのが本システムです。
①はコマンドを列方向に展開し、行数を抑えることで解決を試みました。
②はコマンドを罫線+タイトルでグループ分けし、コマンドを構造化することでプレイヤーが得る情報量を整理しました。
③は色付けを行うことでグループ毎のイメージを作ったのも理解の一助としました。

===================================
4.工夫した点
===================================
・重要度が高いコマンド群から順に並べるようにしました。
